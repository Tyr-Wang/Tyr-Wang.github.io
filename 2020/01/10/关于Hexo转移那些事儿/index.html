放假回家，blogs怎么办/笑哭

两个电脑之间的同步和备份一直是一个老大难。又赶上放假回家……

参考了前人经验，我就不再赘述。

两个方向

  1. 手动复制文件，手动安装依赖，添加ssh密钥，好像在骗GitHub我没换电脑呀——简单粗暴易执行；
  2. 利用git原本的功能，手动pull、push

当然选择了第一个啦，记录一下

好像就似从头~再来

安装必要软件

安装 Git、node JS

在 github 添加新电脑产生的密钥

右键 -> Git Bash Here，设置用户名和邮箱：

创建 SSH 密匙：

输入 ssh-keygen -t rsa -C "GitHub 邮箱"，然后一路回车。

添加密匙：

进入 [C:\Users\用户名\.ssh] 目录（要勾选显示“隐藏的项目”），用记事本打开公钥 id_rsa.pub 文件并复制里面的内容。

登陆 GitHub ，进入 Settings 页面，选择左边栏的 SSH and GPG keys，点击 New SSH key。

Title 随便取个名字，粘贴复制的 id_rsa.pub 内容到 Key 中，点击 Add SSH key 完成添加。

验证连接：

打开 Git Bash，输入 ssh -T git@github.com 出现 “Are you sure……”，输入 yes 回车确认。

显示 “Hi xxx! You’ve successfully……” 即连接成功。

为什么是这些，这要从hexo的原理讲起。

观察Github中的文件、hexo g的日志可以发现，渲染生成的页面在public/，刚好与GitHub上完全一致

所以配置本地文件的目的，等价于让hexo g生成的网页正常，即可保证迁移成功

再者，观察初次搭建博客时候的初始化命令，观察生成的文件

简单来说就相当于重新搭建，但是跳过了hexo init步骤，手动初始化，沿用以前的配置、主题、插件，保留之前的博文
