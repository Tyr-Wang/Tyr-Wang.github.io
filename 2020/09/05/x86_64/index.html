x86_64

0x01 64-bit ROP

  • syscall 寄存器传参，完全不用栈，rax指令号，参数rdi, rsi, rdx, rcx, r8, r9, <syscall>, 不需要平栈

  • r8-r15 前缀（REX prefix）40H~4FH+指令表示使用附加寄存器（r8-r15）

  • r12-r15 调用函数时，被调用函数保护现场（callee saved ），所以经常有push/pop r12/…/r15

  • pop r14 = 415e / pop rsi = 5e //可以利用后半段作为rop控制寄存器

  • 同理pop r15 = 415f / pop rdi = 5f

  • ROPgadget 增加预设搜寻长度：
    ROPgadget --binary ./code --depth 100

    一定会有的gadget

  • 控制register

  • 原生小段gadget

  • 控制rax（因为jmp rax 很常见）

    • gets,fgets会回传buffer rax=rdi
      • rdi = &buffer, 所以缺点&buffer必须可写，否则crash
      • 必须向buffer中写入一些，不能设置写入为0，否则gets会crash，fgets return rax = 0
    • strcpy,strncpy同理
      • 可以设置写入长度位0来规避buffer必须可写，strncpy不会crash
    • alarm（call第二次时会ret第一次的alarm的参数）
      • 只能控制unsigned int 即eax，而且必须等第一次，很久

  • 控制rcx（少用）

    • 使用call func 比如strpy，输入字串有时候给rcx
    • 有的syscall，rcx有时是syscall之后下一行的地址，可以call一个syscall(无害的比如alarm)

  • rax写入memory（拿rax值）

    • stack指向mem，然后push_gadget_call,提前修改call的内容为pop_ret

      De-ASLR with rop

      calculate function address without information leakage

      只能送一次payload，做到ret2libc

  • 猜测libc，商务平台有限

  • _dl_runtime_resolve (relro保护时就凉凉）

    从got.plt上取值

    原理简单说就是，从got.plt捞一个base地址，把它和推算出的offset放入内存然后call ptr[base][offset]

  • pop_rsp_r13_r14_r15_ret 转移Stack并pop一系列，出现在csu_init函数pop r12的后半段开始半条指令刚好是pop rsp

  • 需要提前填充寄存器值和返回值在rsp指向的，

  • ret栈上填好leave_ret回来; 并且在一开始就设置好rbp来migrated stack

  • 取出的base在r13上。

  • got.plt 在full relro下是只读的，可以跳到后半段（.got~.data)读完got然后ret刚好到.data可写一点（盖到一些渣渣但不影响:_data…__data_start）
