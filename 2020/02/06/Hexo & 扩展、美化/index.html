Hexo Tricks

Configure

资源（Asset）代表 source 文件夹中除了文章以外的所有文件，例如图片、CSS、JS 文件等。比方说，如果你的Hexo项目中只有少量图片，那最简单的方法就是将它们放在 source/images 文件夹中。

当post_asset_folder: true，Hexo通过 hexo new [layout] <title>命令创建新文章时，自动创建一个同名文件夹。将所有资源放在这个关联文件夹中之后，你可以通过相对路径来引用它们。

修改渲染布局和模板（layout）

每个模板都默认使用 layout 布局，您可在 front-matter 指定其他布局，或是设为 false 来关闭布局功能，您甚至可在布局中再使用其他布局来建立嵌套布局。

使用layout参数改变布局后，并没有取消默认布局的渲染，而是仅仅改变了内嵌body指定的模板；

指定layout: false 也没有效果，所以决定修改ejs默认布局。

修改默认布局：layout.ejs

跳过渲染

文件

对于单个文件（自定义页面）的指定跳过渲染，可以在页面开头以markdown格式加入header，添加属性layout: false

[注意]: 一般主题会默认渲染布局，md文件会被渲染成对应HTML，所以不算完全跳过渲染

目录

全局指定跳过文件、文件夹、递归目录树：全局配置文件_config.yml添加skip_render属性

属性值支持 glob 模式匹配：

  1. 跳过source目录下的 test.html: skip_render: test.html

  2. 跳过source目录下 test 文件夹内所有文件（但不包括子目录及其其中的文件）：skip_render: test/*

  3. 跳过source目录下 test 文件夹内所有文件包括子文件夹以及子文件夹内的文件：skip_render: test/**/*

引用资源

对于单页面跳过，若使用css、js外链，不能放在统一目录下，也会被当作文本渲染（图片可以相对引用）；
所以建议资源放入统一的css根目录下

对于目录跳过，资源即可按照正常的相对路径引用
