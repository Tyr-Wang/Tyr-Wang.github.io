链接库

静态库

代码在lib中

  1. 写好库代码
  2. 编译生成头文件、lib库
  1. 将编译后的lib静态库拷贝到要使用的项目工程中
  2. 在要使用的源代码中引入头文件、添加预编译语句

缺点：

  • 伪模块化，并非真正的解耦合；
    od跟进，发现所有的函数实现都被编译进exe里了，一旦修改库内容，需要重新编译

动态链接库（Dynamic-linked lib）

头文件：

  • extern 关键字表示全局函数，可供其他函数调用
  • “C” 按照C语言方式编译、链接（编译器不会更改函数名，防止按C++编译出现重名函数重载）
  • _declspec(dllexport) 声明此函数为导出函数

使用dll

代码在dll中，lib里写了代码的位置

  • 隐式链接

    1. xxx.dll xxx.lib 放置在同目录下
    2. 调用文件中添加#pragma comment(lib, "xxx.lib")
    3. 加入函数声明：extern "C" _declspec(dllimport) __stdcall int dbl (int x);

  • 显示链接

Handle 系统的内核对象，如文件句柄、线程句柄、进程句柄。
HMODULE 应用程序载入的模块
HINSTANCE == HMODULE （in Win32）
HWND 窗口句柄
都是 unsigned int （DWORD）

  • 隐藏函数名导出——使用.def文件

正常编写库源文件

xxx.def
